# 에디터 컨텍스트가 포커스된 패널을 따라갑니다

> Languages: [English](./en.md) · **한국어**
>
> Related: [#180](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/180), [#199](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/199), [#205](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/205), [#204](https://github.com/yhk1038/claude-code-gui-jetbrains/pull/204), [#207](https://github.com/yhk1038/claude-code-gui-jetbrains/pull/207)

## 새로워진 점

**Claude Code 패널을 여러 개 열어둔 상태**(여러 에디터 탭, 사이드 툴윈도우, 브라우저 탭)에서 에디터 컨텍스트가 어긋났습니다. 라인을 선택하면 파일 뱃지가 **모든** 패널에 붙었고, `Alt+K`(또는 우클릭 "Send to Claude Code")가 엉뚱한 패널에 멘션을 넣거나 새 탭을 계속 열었습니다(이슈 #180, #199, #205).

이제 에디터 컨텍스트는 — 자동 **파일 뱃지**든 명시적 **`Alt+K` 멘션**이든 — **마지막으로 포커스한 패널**로만 갑니다. 채팅을 띄우는 동작도 항상 IDE에 무언가를 여는 대신 그 패널을 따라갑니다.

## 무엇이 달라지나

- **라인 선택 / `Alt+K`** → 파일 뱃지와 멘션이 **지금 보고 있는 패널**에만 붙습니다(모든 패널이 아니라).
- **두 번째 에디터 탭 열기** → 곧바로 연결됩니다. 새 탭에서 "403 · Forbidden"이 더는 뜨지 않습니다(호스트가 이미 인증한 세션을 그대로 재사용).
- **브라우저로 세션 열기**(리모트컨트롤 ⌘-클릭) → 정상 연결되고, 브라우저 탭 두 개도 서로 구분됩니다.
- **툴윈도우 모드 + `Alt+K`** → 누를 때마다 새 에디터 탭을 쌓지 않고, 이미 열린 Claude 패널로 포커스합니다.
- **마지막 포커스가 브라우저 탭인데 `Alt+K`** → 멘션은 그 브라우저 탭으로 가고 IDE는 건드리지 않습니다(툴윈도우가 열리지 않음). 마지막 포커스가 IDE 패널이었으면 그 패널로 포커스하며(툴윈도우가 숨겨져 있었다면 다시 엽니다), 아무것도 열려 있지 않을 때만 새 탭을 엽니다.

## 동작 방식

- **패널마다 하나의 식별자.** 각 패널은 IDE와 Node 백엔드가 공유하는 단일 안정 식별자(`panelId`)로 추적됩니다. 그래서 백엔드가 "포커스된 패널"을 정확한 IDE 탭으로 곧바로 이어줄 수 있습니다. 브라우저 탭은 자기 식별자를 메모리에서 발급하므로, 어떤 방식으로 열리든 탭끼리 충돌하지 않습니다.
- **포커스 이력.** 웹뷰가 포커스(창 포커스, 입력창 포커스, 클릭)를 백엔드에 알리고, 백엔드는 최근 포커스된 패널들의 작은 스택을 유지합니다. 죽은 패널은 자동으로 걸러내므로 맨 위는 항상 살아있는 패널이며, 어떤 패널을 닫으면 그 직전에 포커스했던 패널이 저절로 다시 맨 위로 올라옵니다.
- **범위를 좁힌 전파.** 패널 단위 전파(파일 뱃지 / 멘션)는 이 스택의 맨 위 패널로만 보내되, 아직 포커스를 모를 때는 전체 전파로 물러섭니다 — 그래서 내용이 유실되지 않습니다. `Alt+K` 채팅 노출도 같은 출처를 읽습니다: 대상 IDE 패널을 포커스하거나, 브라우저 탭이면 아무것도 안 하거나, 아무것도 포커스돼 있지 않으면 새 탭을 엽니다.
- **탭 간 세션 재사용.** 실행 단위 인증 토큰을 한 클라이언트의 패널들이 공유하므로 새 탭이 페어링된 세션을 재사용합니다. 별도 클라이언트인 시스템 브라우저는 자기용 단기 페어링 코드를 교환합니다. 페어링은 모든 곳에서 여전히 요구되며 — 자격 재사용만 고쳐졌습니다.

모든 동작은 패널 자신의 포커스 신호와 백엔드 라우팅으로만 이루어집니다 — 공식 SDK나 미문서화 프로토콜에 의존하지 않습니다.
