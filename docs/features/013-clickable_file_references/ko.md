# 채팅의 파일 참조를 클릭해 해당 라인으로 이동

> Languages: [English](./en.md) · **한국어**
>
> 관련: [#183](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/183), [#209](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/209), [#210](https://github.com/yhk1038/claude-code-gui-jetbrains/pull/210)

## 무엇이 달라졌나

Claude가 답변에서 파일을 가리킬 때 — 코드를 설명하거나, 변경을 리뷰하거나,
버그 위치를 짚어줄 때 — 그 참조를 이제 **클릭할 수 있다**. 클릭하면 JetBrains
에디터에서 해당 파일이 열리고, 참조된 **라인(그리고 컬럼이 있으면 컬럼)**으로
캐럿이 바로 이동한다.

이전에는 이 참조들이 동작하지 않았다. Markdown 파일 링크는 오히려 파일 대신
빈 `localhost:<포트>/…` 브라우저 탭을 여는 문제가 있었다(#209).

## 화면에서 보이는 것

두 종류의 참조가 클릭 가능해진다.

- **Claude가 작성한 Markdown 파일 링크**. 예:
  `[ExampleService.java:120-135](./src/main/java/.../ExampleService.java#L120-L135)`.
  클릭하면 120번째 줄에서 파일이 열린다. 일반 웹 링크(`https://…`)는 그대로
  링크로 남아 브라우저에서 열린다.
- **문장 안에 그대로 쓰인 평문 참조**. 예: `src/example/File.java:10`,
  `src/example/File.java:10:5`(라인 **과 컬럼**), `src/example/File.java#L10-L25`.
  이런 표기를 인식해 동일한 클릭 참조로 바꿔준다.
- **`@` 멘션 칩**(`@src/file.ts#L42`)도 이제 라인을 함께 전달한다. 클릭하면
  파일 최상단이 아니라 42번째 줄로 이동한다.

상대경로는 프로젝트 작업 디렉토리를 기준으로 해석되므로, `./src/x.ts`와
`src/x.ts` 모두 올바른 파일을 연다.

## 일부러 건드리지 않는 것

평문 인식은 일반 문장을 깨진 링크로 만들지 않도록 **의도적으로 보수적**이다.

- 참조에는 **경로 구분자**(`/`)가 반드시 있어야 한다. 슬래시 없는 `App.java:10`
  이나 `example.com:8080` 같은 호스트는 링크로 만들지 **않는다**.
- **라인 표기**(`:라인`, `:라인:컬럼`, `#L…`)가 반드시 있어야 한다. 그냥
  `10:30`(시각)이나 라인 없는 `src/app.ts` 언급은 텍스트 그대로 둔다.
- **인라인 코드나 펜스 코드 블록 안**, **기존 링크 안**, **URL 안**의 내용은
  절대 다시 쓰지 않는다. 코드 예시는 원문 그대로 유지된다.

## 동작 방식

- 참조는 IDE 브릿지의 `openFile(path, line, column)`을 호출하는 클릭 요소로
  렌더링된다. 에디터 컨텍스트·`@` 멘션 기능이 이미 쓰는 바로 그 브릿지라서 새
  백엔드 프로토콜이 필요 없다. 라인·컬럼은 1-based이며, 라인이 없으면 파일
  최상단에서 열린다.
- 채팅을 **IDE 세션에서 브라우저 탭으로 열었을 때**도, 참조를 클릭하면 OS 기본
  앱이 아니라 그 **IDE**에서(해당 라인/컬럼으로) 파일이 열린다. 백엔드가 연결된
  IDE가 있으면 그쪽으로 열기를 라우팅하기 때문이다. IDE가 없는 단독 브라우저는
  기존대로 OS 기본 열기로 폴백한다.
- 평문 참조는 웹뷰 전용 전처리 단계에서 링크로 바뀐다. 스캔 전에 코드 스팬,
  기존 링크, URL을 가려낸 뒤 그 결과를 Markdown 링크 렌더러에 그대로 넘긴다.
  덕분에 두 종류의 참조가 하나의 코드 경로를 공유한다.
- Windows 드라이브 경로(`C:\proj\File.java`)는 Markdown 새니타이즈를 안전하게
  통과해 전달되고, 파일을 열 때 복원된다.

이 변경은 웹뷰 전용이다. IDE는 이미 라인 이동을 지원하고 있었으므로, 기존
사용자는 플러그인을 업데이트하기만 하면 이 기능을 얻는다.
