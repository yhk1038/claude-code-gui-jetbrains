# RTL (오른쪽에서 왼쪽) 언어 지원

> Languages: [English](./en.md) · **한국어**
>
> 관련: [#158](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/158)

## 새로운 점

이제 GUI가 오른쪽에서 왼쪽으로 읽는 언어를 위해 화면 전체를 미러링합니다. 새로
생긴 **설정 → General → "RTL(Right-to-left) 켜기"** 토글을 켜면 사이드바, 채팅,
다이얼로그, 드롭다운 등 앱 전체가 페르시아어·아랍어 사용자에게 자연스러운
방향인 오른쪽 기준으로 뒤집힙니다.

이와 함께 **인터페이스 언어**에 새로운 옵션 두 가지가 추가됩니다:

- فارسی (페르시아어)
- العربية (아랍어)

둘 중 하나를 선택하면 RTL이 자동으로 켜지므로 별도 조작이 필요 없습니다.

## 무엇이 보이나

**설정 → General**에서:

- **Interface Language** 바로 아래에 새로운 **"RTL(Right-to-left) 켜기"**
  토글이 생깁니다. 켜면 앱 전체가 즉시 방향을 전환합니다 — 텍스트 정렬,
  방향성을 가진 아이콘(뒤로가기 화살표, 펼침 화살표 등), 드롭다운·메뉴 위치,
  설정 사이드바가 슬라이드로 나타나는 방향까지 모두 미러링됩니다.
- **Interface Language**에 다른 지원 언어와 동일하게 번역된 **فارسی
  (페르시아어)**와 **العربية (아랍어)**가 추가됩니다.

### 언어와 방향은 자동 연동되지만, 최종 결정은 사용자 몫입니다

**Interface Language**를 페르시아어나 아랍어로 바꾸면 RTL이 자동으로 켜지고,
거기서 다시 왼쪽에서 오른쪽으로 읽는 언어로 바꾸면 RTL이 자동으로 꺼집니다 —
언어를 바꿀 때마다 토글을 직접 조작할 필요가 없습니다.

다만 RTL이 언어 선택에 완전히 종속되는 것은 아닙니다. 한 번 설정되고 나면(자동
연동이든 수동 조작이든), 이후에는 토글을 자유롭게 직접 켜고 끌 수 있습니다 —
예를 들어 인터페이스는 영어로 두되 화면만 미러링해서 보거나, 페르시아어를
왼쪽에서 오른쪽 레이아웃으로 사용할 수도 있습니다. 자동 연동은 선택한 언어의
읽기 방향이 실제로 바뀌는 순간에만 작동하며, 그 이후의 수동 선택은 그대로
존중됩니다.

## 그대로 유지되는 것 — 코드, 터미널 출력, 수식

UI 전체를 RTL로 뒤집어도 **코드**는 뒤집히지 않습니다. 코드 블록, 인라인
코드, tool/터미널 출력, 수식(KaTeX)은 페르시아어·아랍어 문장 속에 섞여 있어도
이전과 동일하게 항상 왼쪽에서 오른쪽으로 렌더링됩니다. 스니펫 안에
오른쪽에서-왼쪽으로 읽는 식별자·주석·문자열이 섞여 있어도 구두점·들여쓰기·
연산자 순서가 뒤바뀌지 않습니다 — 메시지에서 모호하면 안 되는 부분은 계속
모호하지 않게 유지됩니다.

## 동작 방식

- 이 토글은 GUI 전용 앱 전역 설정 `uiDirection`(기본값 `"ltr"`)으로 저장되고,
  문서 루트의 `dir` 속성으로 적용됩니다 — 브라우저와 JCEF가 양방향(bidi)
  텍스트·레이아웃에 네이티브로 쓰는 것과 같은 메커니즘이라, 이미 RTL을 인식하는
  다른 요소들과도 자연스럽게 어우러집니다.
- 전체 레이아웃은 고정된 `left`/`right` 대신 CSS 논리 속성(`start`/`end`)으로
  구성되어 있어, 컨테이너 위치·테두리·마진이 방향에 따라 자동으로 뒤집힙니다.
  대신 *물리적* transform을 쓰는 요소들 — 토글 스위치의 슬라이딩 핸들,
  뒤로가기/펼침/화살표 아이콘, 사이드바 드로어의 슬라이드 오프셋 — 은 RTL로
  뒤집힌 값을 명시적으로 함께 갖고 있어서 애니메이션도 올바른 방향으로
  동작합니다.
- 코드 블록·인라인 코드·KaTeX 수식은 명시적으로 왼쪽에서 오른쪽 방향으로
  고정되고 주변 bidi 문맥과 격리되어 있어서, 메시지 어디에 끼어 있든 미러링된
  방향을 물려받지 않습니다.
- 방향 값은 (기존의 다크/라이트 테마 결정과 마찬가지로) 첫 페인트 이전, 부팅
  시퀀스 최대한 초반에 결정되어, 새 에디터 탭이 열릴 때 미러링되지 않은 화면이
  잠깐 보였다가 뒤집히는 깜빡임을 방지합니다.

## 컨트리뷰터를 위한 참고

RTL은 **인터페이스 언어(i18n)** ([기능 009](../009-interface_language_i18n/ko.md))와는
별개의 레이어입니다 — 원칙적으로 언어는 RTL 없이 번역될 수 있고, RTL도 언어와
무관하게 켜고 끌 수 있습니다. 페르시아어·아랍어는 기본값에서 두 가지를 동시에
켜는 것일 뿐입니다.
