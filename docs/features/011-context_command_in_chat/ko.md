# 채팅에서 컨텍스트 사용량 표시

> Languages: [English](./en.md) · **한국어**
>
> 관련: [#196](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/196), [#198](https://github.com/yhk1038/claude-code-gui-jetbrains/pull/198)

## 새로운 점

`/context`를 채팅에서 실행하면 **아무것도 표시되지 않던** 문제가 있었습니다 —
응답이 조용히 사라졌습니다(이슈 #196). 이제 정상 동작하며, 응답을 순정
`claude` 터미널 출력과 최대한 똑같이 재현한 **컨텍스트 사용량 카드**로
보여줍니다. 같은 격자, 같은 카테고리 분석, 같은 트리를 GUI 안에서 그대로.

## 표시 내용

`/context`를 보내면 카드에 다음이 나옵니다:

- **헤더**: 모델 표시 이름(예: `Opus 4.8 (1M context)`)과 그 아래 모델 id,
  그리고 토큰 요약(`28.2k/1m tokens (3%)`).
- **사용량 격자**: 컨텍스트 창을 유니코드 셀 매트릭스로 그립니다 — 카테고리
  색이 적용된 `⛁` 채움 셀, 여유 공간은 `⛶` 빈 셀. 터미널과 동일하게 정렬.
- **카테고리 분석**: *Estimated usage by category* 아래에 `⛁ System prompt:
  2.6k tokens (0.3%)` 형태의 인라인 항목을, CLI가 준 순서 그대로.
- **트리 형태의 상세 섹션**: MCP Tools, Custom Agents, Memory Files, Skills —
  각각 경로 부제(`· /mcp`, `· .claude/agents/`, `· /memory`, `· /skills`)와
  소스별 그룹, 토큰 비용은 회색 텍스트로.
- **MCP Tools**는 서버별로 묶여 **기본 접힘** 상태이며 서버마다 도구 개수를
  보여줍니다. 펼치면 각 도구의 토큰 비용까지 확인할 수 있습니다.

### 원본 응답 보기

카드 헤더의 **원본 응답 보기(Show original response)** 토글로 편집되지 않은
CLI 원본 마크다운으로 전환할 수 있습니다. 원본을 한 번의 클릭으로 볼 수 있고,
향후 상위(CLI) 출력 형식이 바뀌더라도 카드가 자연스럽게 대응할 수 있습니다.

## 동작 방식

- `/context`는 Claude Code CLI가 자체적으로 렌더링하는 클라이언트 사이드
  명령입니다. 우리는 CLI를 stream-json 모드로 실행하고 `/context`를 일반
  메시지로 전달합니다 — 미문서화 프로토콜이나 별도 호출 없이. CLI는 완성된
  마크다운 응답(섹션별 테이블)을 돌려줍니다.
- 일반 대화와 달리 이런 로컬 명령은 **부분 스트리밍 이벤트 없이** `assistant`
  결과를 내보내는데, 이때 렌더링 경합이 드러나 내용이 유실됐습니다. React가
  업데이트를 반영하기 전에 스트리밍 대상을 미리 캡처하도록 고쳐 해결했습니다.
- 카드는 그 마크다운을 **웹뷰에서 파싱**해 순정 TUI 레이아웃으로 재구성합니다.
  백엔드와 전달 계층은 편집하지 않습니다 — 원본 마크다운은 그대로 전달되고(원본
  데이터 보존), "원본 응답 보기"는 CLI가 보낸 그대로를 렌더링합니다. 파싱이
  실패하면 카드는 평문 마크다운으로 폴백합니다.

이로써 `/context`는 완전히 CLI 동등하게 동작합니다 — 터미널에서 보던 것을 이제
GUI에서도, 공식 SDK나 미문서화 프로토콜이 아니라 CLI 자신의 출력으로부터
재구성해서 보여줍니다.
