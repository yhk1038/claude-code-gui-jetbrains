# 에디터 컨텍스트 태그 (Editor Context Tag)

> 언어: [English](./en.md) · **한국어**
>
> 관련: [이슈 #122](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/122) · [PR #133](https://github.com/yhk1038/claude-code-gui-jetbrains/pull/133)

## 새로운 기능

이제 Claude Code with GUI는 **지금 보고 있는 파일이나 선택한 코드를 채팅 입력창 바로 위에 컨텍스트 태그로 자동 표시**합니다. 메시지를 보내면 그 파일이나 선택 영역이 프롬프트와 함께 Claude에게 전달되므로, "지금 보고 있는 파일"을 일일이 설명하거나 코드를 직접 붙여넣을 필요가 없습니다.

이는 Cursor / VS Code의 Claude Code 확장이 제공하던 경험을 JetBrains IDE에 그대로 가져온 것입니다.

## 동작 방식

- **파일 탭을 열거나 전환** → 입력창 하단(footer)에 파일 이름이 적힌 태그가 나타납니다 (예: `📄 settings.json`). Claude는 다음 메시지에서 그 파일을 컨텍스트로 참고합니다.
- **에디터에서 코드를 선택** → 태그가 라인 범위로 바뀌고 (예: `settings.json:42-51`), 선택한 코드가 메시지와 함께 전달됩니다.
- **태그를 클릭하면 포함/제외를 토글**합니다. 포함 상태에서는 파일 아이콘이, 클릭해서 제외하면 눈 가림 아이콘(`👁️‍🗨️`)이 표시됩니다. 제외하면 그 파일/선택은 전송되지 않습니다. 다시 누르면 포함됩니다.
- 태그는 Claude Code 탭이 열려 있을 때만 나타나며, Claude 패널 자체를 가리키지는 않습니다.

평소처럼 입력을 계속하면 됩니다 — 태그는 조용히 백그라운드에서 동작하며, 실제로 메시지를 보낼 때만 컨텍스트를 첨부합니다.

> **참고:** 이 기능은 기존 **Alt+K** 동작과 별개입니다. Alt+K는 필요할 때 파일 경로를 `@멘션`으로 입력창에 *삽입*하고, 컨텍스트 태그는 현재 보고 있는 대상을 *자동으로* 추적하며 토글할 수 있게 합니다. 둘은 함께 쓸 수 있습니다.

## .gitignore 존중 (선택, 기본 꺼짐)

열어 보는 파일 중에는 `.env`처럼 `.gitignore`에 등록된 민감한 파일이 있을 수 있고, 그 내용이 기기 밖으로 나가는 걸 원하지 않을 수 있습니다.

**Settings → General → `Respect .gitignore for editor context`** 를 켜면, 보고 있는 파일이 gitignore 대상일 때 **경로만** Claude에게 공유되고 **파일 내용은 제외**됩니다. 태그는 그대로 표시되어 어떤 파일인지 인지할 수 있습니다.

이 설정은 **기본적으로 꺼져 있으므로**, 직접 켜기 전에는 동작이 달라지지 않습니다. 다른 설정처럼 범위별(전역 / 프로젝트)로 설정할 수 있습니다.

## 알아두기

- 슬래시 명령(`/`로 시작하는 메시지)에는 에디터 컨텍스트가 실리지 않으므로, 명령 동작은 예측 가능하게 유지됩니다.
- 같은 파일/선택이 바뀌지 않았다면 매 메시지마다 다시 첨부되지 않습니다 — 위치나 선택을 바꾸기 전까지 한 번만 전송됩니다.
