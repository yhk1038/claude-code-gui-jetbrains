# 인터페이스 언어 (UI 번역)

> Languages: [English](./en.md) · **한국어**
>
> 관련: [#141](https://github.com/yhk1038/claude-code-gui-jetbrains/issues/141), [#160](https://github.com/yhk1038/claude-code-gui-jetbrains/pull/160)

## 새로운 점

이제 GUI 전체를 원하는 언어로 표시할 수 있습니다. 설정 화면, 채팅 화면, 도구 카드,
다이얼로그, 세션 목록, 알림까지 — 앱이 그리는 모든 것이 **10개 언어**로 번역됩니다.

새로 생긴 **설정 → General → Interface Language** 항목으로 조절하며, 이것은
**Claude의 응답 언어(기존 "Language" 설정)와 분리**되어 있습니다. 서로 다른 두 가지입니다.

- **Language** — Claude가 *응답하는* 언어. Claude 설정 파일에서처럼 자유 텍스트라
  어떤 언어든 직접 입력할 수 있습니다.
- **Interface Language** — *앱 UI 자체가* 표시되는 언어. 번역을 제공하는 언어들의
  드롭다운입니다.

## 지원 인터페이스 언어

- English (영어)
- 한국어
- 日本語 (일본어)
- 简体中文 (중국어 간체)
- 繁體中文 (중국어 번체)
- Español (스페인어)
- Français (프랑스어)
- Deutsch (독일어)
- Português (포르투갈어)
- Русский (러시아어)

## 무엇이 보이나

**설정 → General**에서:

- **Language**는 이제 텍스트 입력창입니다. 한 번도 설정하지 않았다면 "English"
  플레이스홀더와 함께 비어 있고(Claude 기본값), 값이 있었다면 그대로 표시되며
  플러그인을 업데이트해도 **덮어쓰지 않습니다**.
- 바로 아래 **Interface Language** 드롭다운이 새로 생겼습니다. 항목은 각 언어의
  현지어 이름(한국어, 日本語, 简体中文, 繁體中文, …)으로 표기됩니다. 하나 고르면
  UI가 즉시 전환되어 사이드바/툴 윈도우·채팅·다이얼로그 등 모든 화면이 따라갑니다.

인터페이스 언어를 고르지 않으면 UI는 기본적으로 **영어**로 유지됩니다(응답 언어를
따라가지 않습니다).

## 동작 방식

- 번역은 **react-i18next** 기반입니다. 앱의 각 영역이 언어별 문자열 카탈로그를
  가지며, 아직 번역되지 않은 항목은 영어로 폴백되어 앱은 항상 정상 동작합니다.
- 인터페이스 언어는 GUI 전용 설정으로, Claude의 동작이나 `settings.json`의 응답
  언어를 **건드리지 않습니다**. 한쪽을 바꿔도 다른 쪽은 바뀌지 않습니다.
- 중국어 번체와 간체는 **별도 로케일**(`繁體中文` / `简体中文`)로 제공됩니다.
- 새 언어 추가는 컨트리뷰터가 쉽게 할 수 있도록, 저장소에 번역 파일 생성·언어
  등록 절차를 안내하는 `add-locale` 스킬이 포함되어 있습니다.
  [`.claude/skills/add-locale/SKILL.md`](../../../.claude/skills/add-locale/SKILL.md) 참고.
